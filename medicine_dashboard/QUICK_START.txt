⚡ QUICK START (3 EASY STEPS)
==============================

STEP 1: Upload Arduino Code (2 minutes)
────────────────────────────────────────
1. Download Arduino IDE from https://www.arduino.cc/en/software
2. Open Arduino IDE
3. Open your project file: medicine_reminder_arduino.ino
4. Connect Arduino to PC with USB cable
5. Tools → Board → Select "Arduino Uno" (or your board)
6. Tools → Port → Select COM port (probably COM3 or COM4)
7. Click Upload button (→|) or press Ctrl+U
8. Wait for "Done uploading" message

✓ DONE! Arduino is ready.


STEP 2: Install Python Packages (1 minute)
──────────────────────────────────────────
1. Open Command Prompt (Windows: Win+R → cmd)
2. Navigate to project folder:
   cd C:\path\to\medicine_reminder

3. Run:
   pip install -r requirements.txt

   (If that fails, try:)
   pip install Flask pyserial

✓ DONE! Python is ready.


STEP 3: Start the Web Server (30 seconds)
─────────────────────────────────────────
1. Double-click start.bat
   OR
   Open Command Prompt and type:
   python app.py

2. You should see:
   ============================================================
   Smart Medicine Reminder System - Web Interface
   ============================================================
   Open http://localhost:5000 in your browser
   ============================================================

3. Open browser and go to: http://localhost:5000

4. Dashboard loads! ✓

🔧 TROUBLESHOOTING IN 60 SECONDS
=================================

BUZZER NOT WORKING?
→ Check wiring: positive pin to Arduino Pin 9, negative to GND
→ Open Arduino Serial Monitor and type "ALARM" to test directly

LED NOT BLINKING?
→ Check LED polarity (long leg = positive)
→ Check resistor (220Ω) is connected
→ Try swapping wires to see if LED lights

NO "Arduino Connected" MESSAGE?
→ Check USB cable
→ Check port: Device Manager → Ports → Should see "Arduino" or "COM#"
→ Try different USB port
→ Restart Arduino (unplug 5 sec, plug back in)
→ Verify port in Device Manager matches port in Arduino IDE

WEB PAGE WON'T OPEN?
→ Flask must be running (check Command Prompt window is open)
→ Try http://localhost:5000 (not https://)
→ Close and restart Flask app
→ Check port 5000 is not used by another program

MEDICINE TIMES NOT SAVING?
→ Check flask server console for error messages
→ Ensure folder has write permissions
→ Try adding time again
→ Restart Flask app

📊 WHAT HAPPENS AUTOMATICALLY
==============================

Every second:
  1. Flask checks if current time matches any medicine time
  2. If match found:
     a. Sends "ALARM" command to Arduino
     b. Arduino activates buzzer (beeps 3x)
     c. Arduino LED blinks continuously
     d. Web dashboard shows "ALARM ACTIVE"
     e. Browser shows alert notification

User clicks "OK" or "Stop":
  1. Sends "STOP" command to Arduino
  2. Buzzer and LED turn off
  3. Dashboard returns to "Idle"
  4. System waits for next medicine time

System won't trigger same time twice
  • Only triggers once per scheduled time
  • Resets when time changes

🎓 UNDERSTANDING THE SYSTEM
============================

Hardware Level (Arduino):
  • Receives text commands via serial
  • "ALARM" → Beep buzzer, blink LED
  • "STOP" → Turn off buzzer and LED
  • "STATUS" → Report current state

Software Level (Flask):
  • Runs time-check loop in background
  • Checks time every 1 second
  • Serves web dashboard
  • Communicates with Arduino via COM port

User Level (Web Dashboard):
  • View/manage medicine times
  • Monitor alarm status
  • Test hardware
  • Trigger alarm manually
